## 简介

火瞳系统 AstrBot 插件是一个智能火灾调查助手，可以通过聊天机器人平台（QQ、Telegram等）访问火瞳知识图谱系统。

### 主要功能
- 🔍 **智能查询**: 查询火灾事故、隐患、后果等信息
- 📄 **文档上传**: 上传火灾调查报告并自动分析
- 📊 **统计查看**: 查看系统统计信息
- 💬 **多轮对话**: 支持上下文理解的连续对话

## 安装指南

### 前置要求
- Python 3.8+
- AstrBot 框架已安装
- 火瞳后端系统运行中

### 安装步骤

#### 1. 下载插件
```bash
# 克隆或下载插件到 AstrBot 插件目录
cd /path/to/astrbot/plugins
git clone <repository-url> astrbot-plugin-fireeye
```

#### 2. 安装依赖
```bash
cd astrbot-plugin-fireeye
pip install -r requirements.txt
```

#### 3. 配置插件
编辑 `config.yaml` 文件：
```yaml
fire_eye:
  api_url: "http://localhost:8000"  # 火瞳后端地址
  api_key: "your-api-key-here"      # API 密钥
  timeout: 30
  retries: 3
```

#### 4. 重启 AstrBot
```bash
# 重启 AstrBot 以加载插件
astrbot restart
```

## 配置说明

### 完整配置文件示例

```yaml
# 火瞳后端配置
fire_eye:
  api_url: "http://localhost:8000"
  api_key: "your-api-key-here"
  timeout: 30
  retries: 3

# 会话配置
session:
  ttl: 1800  # 会话过期时间（秒）

# 请求队列配置
request_queue:
  max_concurrent: 5   # 最大并发请求数
  max_queue_size: 20  # 队列最大长度

# 意图检测（可选）
intent_detection:
  enabled: false

# 文件上传配置
file_upload:
  retain_files_on_server: false
  temp_dir: "./temp"
  max_file_size: 10485760  # 10MB

# 日志配置
logging:
  level: "INFO"
  file: "./logs/fireeye.log"
  max_size: "10 MB"
  backup_count: 5
```

### 配置项说明

#### fire_eye 配置
- `api_url`: 火瞳后端 API 地址
- `api_key`: API 认证密钥（从后端管理员获取）
- `timeout`: 请求超时时间（秒）
- `retries`: 失败重试次数

#### session 配置
- `ttl`: 会话过期时间，默认 1800 秒（30分钟）

#### request_queue 配置
- `max_concurrent`: 同时处理的最大请求数
- `max_queue_size`: 等待队列的最大长度

#### file_upload 配置
- `retain_files_on_server`: 是否在服务器保留上传的文件
- `temp_dir`: 临时文件目录
- `max_file_size`: 最大文件大小（字节）

```

1. 【FireEvent】3.21某工厂火灾事故
   📅 日期: 2024-03-21
   📍 地点: 某市某区某工厂

2. 【FireHazard】电气线路老化
   ⚠️ 风险等级: 高

3. 【FireConsequence】财产损失
```

#### 多轮对话

### 2. 上传命令

#### 上传文档
```
/火瞳上传
/ht上传
```

**功能**: 上传火灾调查报告并自动分析

**使用步骤**:
1. 发送 `/火瞳上传` 命令
2. 按照提示发送文件（PDF、DOCX 或 TXT）
3. 等待系统分析（通常需要 30-60 秒）
4. 查看分析结果

**支持的文件格式**:
- PDF (.pdf)
- Word 文档 (.docx)
- 文本文件 (.txt)

**文件大小限制**: 最大 10MB

**示例对话**:
```
用户: /火瞳上传
机器人: 📤 请发送要上传的文件
      支持格式: PDF、DOCX、TXT
      最大大小: 10MB

      📋 任务 ID: task-abc123
      ⏱️ 预计时间: 60 秒
      
      正在处理中，请稍候...

[60秒后]
机器人: ✅ 分析完成
      
      📊 提取结果:
        • 事件链: 3 个
        • 实体: 25 个
        • 关系: 40 个
      
      🔗 事件链:
        1. 电气线路老化 → 短路 → 起火
        2. 消防设施不完善 → 火势蔓延
        3. 人员疏散不及时 → 人员伤亡
```

## 常见问题

### Q1: 如何获取 API 密钥？
**A**: 联系火瞳系统管理员获取 API 密钥。每个用户或组织都有唯一的密钥。

### Q2: 会话会保持多久？
**A**: 默认情况下，会话在 30 分钟无活动后自动过期。可以在配置文件中调整 `session.ttl` 参数。

### Q3: 支持哪些聊天平台？
**A**: 插件支持 AstrBot 框架支持的所有平台，包括：
- QQ
- Telegram
- Discord
- 微信（需要额外配置）

### Q4: 上传的文件会被保存吗？
**A**: 默认情况下，文件在分析完成后会被自动删除。如果需要保留，可以在配置中设置 `file_upload.retain_files_on_server: true`。

### Q5: 查询结果太多怎么办？
**A**: 系统默认只显示前 5 条结果。可以使用更具体的查询词来缩小范围。

### Q6: 可以同时上传多个文件吗？
**A**: 目前每次只能上传一个文件。如需上传多个文件，请分别上传。

### Q7: 如何查看任务处理进度？
**A**: 上传文件后，系统会自动轮询任务状态并更新进度。无需手动查询。

### Q8: 支持语音命令吗？
**A**: 目前不支持语音命令，但这是未来计划中的功能。

## 故障排除

### 问题 1: 插件无法加载

**症状**: AstrBot 启动时提示插件加载失败

**可能原因**:
- 依赖未安装
- 配置文件格式错误
- Python 版本不兼容

**解决方案**:
```bash
# 1. 检查依赖
pip install -r requirements.txt

# 2. 验证配置文件
python -c "import yaml; yaml.safe_load(open('config.yaml'))"

# 3. 检查 Python 版本
python --version  # 应该是 3.8+
```

### 问题 2: 无法连接到后端

**症状**: 查询时提示"连接失败"或"超时"

**可能原因**:
- 后端服务未启动
- API 地址配置错误
- 网络问题

**解决方案**:
```bash
# 1. 检查后端服务
curl http://localhost:8000/health

# 2. 验证配置
cat config.yaml | grep api_url

# 3. 测试网络连接
ping <backend-host>
```

### 问题 3: API 密钥无效

**症状**: 提示"认证失败"或"无效的 API 密钥"

**解决方案**:
1. 确认 API 密钥正确无误
2. 检查密钥是否已过期
3. 联系管理员重新获取密钥

### 问题 4: 文件上传失败

**症状**: 上传文件时提示错误

**可能原因**:
- 文件格式不支持
- 文件大小超限
- 临时目录权限问题

**解决方案**:
```bash
# 1. 检查文件格式
file your-document.pdf

# 2. 检查文件大小
ls -lh your-document.pdf

# 3. 检查临时目录权限
ls -ld ./temp
chmod 755 ./temp
```

### 问题 5: 查询结果不准确

**症状**: 查询返回的结果与预期不符

**可能原因**:
- 查询词不够具体
- 知识图谱数据不完整
- 上下文理解错误

**解决方案**:
1. 使用更具体的查询词
2. 尝试不同的表达方式
3. 开始新会话（清除上下文）

### 问题 6: 系统响应慢

**症状**: 命令执行时间过长

**可能原因**:
- 后端负载高
- 网络延迟
- 请求队列满

**解决方案**:
1. 检查后端系统状态
2. 稍后重试
3. 联系管理员检查系统性能
