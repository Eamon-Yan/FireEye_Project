## 🚀 系统启动

### 前置条件
- Docker Desktop 已安装并运行
- Python 3.11+ 已安装
- Node.js 18+ 已安装

### 启动步骤

#### 1. 启动数据库服务（Docker）
```bash
cd fire-eye-system
docker-compose up -d
```
这将启动：
- Neo4j 图数据库（端口 7474/7687）
- Redis 缓存（端口 6379）

#### 2. 启动后端服务
```bash
cd backend
start-backend.bat
```
后端将运行在：`http://localhost:8000`

#### 3. 启动前端服务
```bash
cd frontend
start-frontend.bat
```
前端将运行在：`http://localhost:8080`

## 👤 用户操作流程

### 流程图
```
开始 → 访问主页 → 上传文档 → 查看处理结果 → 浏览图谱 → 查看节点详情 → 导出数据
```

### 详细步骤

#### 步骤 1：访问系统主页
1. 打开浏览器访问：`http://localhost:8080`
2. 主页显示图谱可视化界面（如果已有数据）

**主页功能：**
- 🔥 FireEye Graph 标题
- 🏠 返回主页按钮（左上角）
- 🔍 搜索框：搜索节点
- 📊 统计数据：显示节点数和链接数
- 🔄 刷新按钮：重新加载图谱数据
- 💾 导出按钮：导出图谱为 JSON
- 📖 图例：显示节点类型（左下角）

---

#### 步骤 2：上传火灾调查报告

**方式 1：从主页导航**
1. 点击页面上的"上传文档"链接
2. 跳转到上传页面：`http://localhost:8080/upload`

**方式 2：直接访问**
- 直接访问：`http://localhost:8080/upload`

**上传页面功能：**
- 📁 文件选择器（支持拖拽上传）
- 📄 支持格式：PDF、Word (.docx, .doc)、TXT
- ⬆️ 上传按钮
- ⬅️ 返回首页按钮

**上传操作：**
1. 点击"选择文件"或拖拽文件到上传区域
2. 选择火灾调查报告文件
3. 点击"上传"按钮
4. 等待系统处理（显示进度条）

---

#### 步骤 3：查看处理结果

上传成功后，页面会显示处理结果：

**文档信息卡片：**
- 文件名
- 文档 ID
- 文件类型
- 章节数量

**处理统计卡片：**
- 原始事件链数量
- 有效事件链数量
- 验证通过率
- 平均置信度

**事件链列表：**
- 显示前 10 个提取的事件链
- 格式：`源节点 → 关系 → 目标节点`
- 显示置信度百分比
- 显示上下文信息

**操作按钮：**
- 🗺️ **查看图谱**：跳转到图谱可视化页面
- 📤 **继续上传**：清除结果，继续上传新文档

---

#### 步骤 4：浏览知识图谱

点击"查看图谱"后，进入图谱可视化页面：

**图谱界面布局：**

**顶部工具栏（居中）：**
- 🏠 返回主页按钮
- 🔥 FireEye Graph 标题
- 🔍 搜索框：输入关键词搜索节点
- 📊 统计数据：Nodes（节点数）、Links（链接数）
- 🔄 刷新按钮：重新加载图谱
- 💾 导出按钮：导出图谱数据为 JSON

**左下角图例：**
- 🔴 Fire Event（火灾事件）- 玫瑰红
- 🟠 Hazard（危险因素）- 琥珀橙
- 🟣 Consequence（后果）- 紫罗兰

**中央图谱区域：**
- 力导向图布局
- 节点：圆形，颜色根据类型区分
- 连线：箭头表示因果关系
- 标签：节点下方显示描述

**交互操作：**
- 🖱️ **拖拽节点**：移动节点位置
- 🔍 **滚轮缩放**：放大/缩小图谱
- 🖱️ **拖拽背景**：平移视图
- 👆 **点击节点**：显示节点详情面板

---

#### 步骤 5：查看节点详情

点击任意节点后，右侧会弹出详情面板：

**详情面板内容：**

**1. 节点类型标签**
- 彩色标签显示节点类型
- 颜色与图例一致

**2. 描述信息**
- 节点的详细描述文本

**3. 属性列表**（如果有）
- 显示节点的所有属性
- 格式：`属性名: 属性值`

**4. 连接关系**
- 显示与该节点相连的其他节点（最多 5 个）
- 每个连接显示：
  - 连接节点的描述
  - 关系类型
  - 方向（→ 出站，← 入站）
- 如果超过 5 个，显示剩余数量

**操作：**
- ❌ 点击右上角 X 按钮关闭详情面板
- 点击其他节点切换查看

---

#### 步骤 6：搜索节点

**搜索功能：**
1. 在顶部工具栏的搜索框输入关键词
2. 按 Enter 或点击搜索图标
3. 系统会筛选包含关键词的节点
4. 图谱更新显示搜索结果

**搜索范围：**
- 节点描述
- 节点类型
- 节点属性

---

#### 步骤 7：导出图谱数据

**导出操作：**
1. 点击顶部工具栏的导出按钮（💾）
2. 系统自动下载 JSON 文件
3. 文件名格式：`fire-eye-graph-{时间戳}.json`

**导出内容：**
```json
{
  "nodes": [
    {
      "id": "节点ID",
      "type": "节点类型",
      "description": "节点描述",
      "properties": { ... }
    }
  ],
  "links": [
    {
      "source": "源节点ID",
      "target": "目标节点ID",
      "relation": "关系类型",
      "confidence": 0.95
    }
  ]
}
```

## 🏗️ 系统架构

### 三层架构

```
┌─────────────────────────────────────────┐
│         前端层 (Frontend)                │
│  React + Next.js + TypeScript           │
│  - 文档上传界面                          │
│  - 图谱可视化 (react-force-graph-2d)    │
│  - 节点详情面板                          │
└─────────────────┬───────────────────────┘
                  │ HTTP/REST API
┌─────────────────▼───────────────────────┐
│         后端层 (Backend)                 │
│  Python + FastAPI + Pydantic            │
│  - 文档解析服务                          │
│  - LLM 事件链抽取                        │
│  - 术语归一化                            │
│  - 逻辑校验                              │
│  - 图谱查询 API                          │
└─────────────────┬───────────────────────┘
                  │ Cypher Query
┌─────────────────▼───────────────────────┐
│         数据层 (Database)                │
│  Neo4j (图数据库) + Redis (缓存)         │
│  - 节点存储 (FireEvent, Hazard, etc.)   │
│  - 关系存储 (因果链)                     │
│  - 图查询优化                            │
└─────────────────────────────────────────┘
```

### 端口分配
- **前端**：8080
- **后端**：8000
- **Neo4j HTTP**：7474
- **Neo4j Bolt**：7687
- **Redis**：6379

## 🔄 数据处理流程

### 完整数据流

```
1. 文档上传
   ↓
2. 文档解析
   - PDF/Word/TXT 解析
   - 章节分割
   ↓
3. LLM 事件链抽取
   - 调用大语言模型
   - 识别因果关系
   - 提取事件三元组 (源, 关系, 目标)
   ↓
4. 术语归一化
   - 标准化术语
   - 映射到预定义词汇表
   ↓
5. 逻辑校验
   - 验证事件链合理性
   - 计算置信度
   - 过滤低质量数据
   ↓
6. 存储到 Neo4j
   - 创建节点 (FireEvent, Hazard, Consequence)
   - 创建关系 (因果链)
   - 建立索引
   ↓
7. 图谱可视化
   - 查询图数据
   - 力导向布局
   - 交互式展示
```

### 节点类型

| 类型 | 说明 | 颜色 | 示例 |
|------|------|------|------|
| **FireEvent** | 火灾事件 | 🔴 玫瑰红 | 电气火灾、森林火灾 |
| **Hazard** | 危险因素 | 🟠 琥珀橙 | 电线老化、易燃物堆积 |
| **Consequence** | 后果 | 🟣 紫罗兰 | 人员伤亡、财产损失 |

### 关系类型
- **导致** (leads_to)
- **引发** (triggers)
- **加剧** (aggravates)
- **产生** (produces)

## ❓ 常见问题

### Q1: 上传文档后没有反应？
**解决方案：**
1. 检查后端服务是否运行：访问 `http://localhost:8000/docs`
2. 检查文件格式是否支持（PDF、Word、TXT）
3. 查看浏览器控制台是否有错误信息
4. 检查后端日志：`backend/logs/`

### Q2: 图谱显示空白？
**解决方案：**
1. 确认已上传文档并处理成功
2. 点击刷新按钮重新加载
3. 检查 Neo4j 是否运行：访问 `http://localhost:7474`
4. 检查后端 API：`http://localhost:8000/api/v1/graph/query`

### Q3: 节点详情面板不显示？
**解决方案：**
1. 确保点击的是节点，不是空白区域
2. 刷新页面重试
3. 检查浏览器控制台是否有 JavaScript 错误

### Q4: 搜索功能不工作？
**解决方案：**
1. 确保输入了搜索关键词
2. 按 Enter 键或点击搜索图标
3. 检查后端 API 是否正常

### Q5: 如何清空图谱数据？
**解决方案：**
```bash
# 方法 1：重启 Neo4j 容器
docker-compose restart neo4j

# 方法 2：使用 Neo4j 浏览器清空
# 访问 http://localhost:7474
# 执行 Cypher 查询：MATCH (n) DETACH DELETE n
```

### Q6: 端口被占用怎么办？
**解决方案：**
1. 修改 `docker-compose.yml` 中的端口映射
2. 修改 `backend/.env` 中的 `PORT` 配置
3. 修改 `frontend/package.json` 中的启动端口
