# 火瞳系统 - 启动指南

## 📋 前置要求

确保已安装：
- Docker Desktop（用于 Neo4j 和 Redis）
- Python 3.8+（用于后端）
- Node.js 16+（用于前端）

---

## 🚀 启动步骤

### 第一步：启动数据库服务（Docker）

打开 PowerShell 或 CMD，进入项目目录：

```bash
cd fire-eye-system
docker-compose up -d neo4j redis
```

验证服务是否启动：
```bash
docker ps
```

你应该看到 `fire-eye-neo4j` 和 `fire-eye-redis` 两个容器在运行。

---

### 第二步：启动后端服务

**方法1：使用批处理文件（推荐）**

双击运行：
```
fire-eye-system/backend/start-backend.bat
```

**方法2：手动命令**

打开新的 PowerShell/CMD 窗口：
```bash
cd fire-eye-system/backend
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

验证后端是否启动：
- 访问 http://localhost:8000/health
- 应该看到 `{"status":"healthy",...}`

---

### 第三步：启动前端服务

**方法1：使用批处理文件（推荐）**

双击运行：
```
fire-eye-system/frontend/start-frontend.bat
```

**方法2：手动命令**

打开新的 PowerShell/CMD 窗口：
```bash
cd fire-eye-system/frontend
npm run dev -- -p 8080
```

等待编译完成，看到 `✓ Ready in X.Xs`

---

## 🌐 访问应用

启动完成后，在浏览器访问：

- **前端主页**: http://localhost:8080
- **图谱页面**: http://localhost:8080/graph
- **文档上传**: http://localhost:8080/upload
- **后端API**: http://localhost:8000
- **Neo4j浏览器**: http://localhost:7474

---

## 🛑 停止服务

### 停止前端和后端
在运行它们的终端窗口按 `Ctrl + C`

### 停止 Docker 服务
```bash
cd fire-eye-system
docker-compose down
```

## ⚠️ 常见问题

### 问题1：端口被占用
如果端口 3000 或 3001 被占用，前端使用端口 8080。

### 问题2：浏览器显示旧样式
- 按 `Ctrl + Shift + R` 强制刷新
- 或使用无痕模式打开

### 问题3：后端连接失败
检查 Docker 容器是否运行：
```bash
docker ps
```

### 问题4：前端编译错误
清除缓存重新启动：
```bash
cd fire-eye-system/frontend
rmdir /s /q .next
npm run dev -- -p 8080
```

## 🎯 开发建议

1. **保持三个终端窗口打开**：
   - 窗口1：后端服务
   - 窗口2：前端服务
   - 窗口3：执行其他命令

2. **修改代码后**：
   - 后端：自动重载（uvicorn --reload）
   - 前端：自动热更新（Next.js）
   - 只需刷新浏览器即可看到更改

3. **查看日志**：
   - 后端日志：在后端终端窗口
   - 前端日志：在前端终端窗口 + 浏览器控制台（F12）
