# 火瞳系统 - 启动指南

## 📋 前置要求

确保已安装：
- Docker Desktop（用于 Neo4j 和 Redis）
- Python 3.8+（用于后端）
- Node.js 16+（用于前端）

---

## 🚀 启动步骤

### 第一步：启动全部服务（Docker）

打开 PowerShell 或 CMD，进入项目目录：

```bash
cd fire-eye-system
docker compose up -d
```

验证服务是否启动：
```bash
docker ps
```

你应该看到至少这些容器在运行：
- `fire-eye-neo4j`
- `fire-eye-redis`
- `fire-eye-backend`
- `fire-eye-frontend`

如果 AstrBot 也需要联调，还应看到：
- `astrbot`

---

### 第二步：确认服务状态

检查后端健康状态：
```bash
curl http://localhost:8000/health
```

或直接在浏览器打开：
- http://localhost:8000/health

应该看到类似：
```json
{"status":"healthy",...}
```

如果后端刚启动时连接不到 Neo4j，请先等待 Neo4j 变为 healthy，再执行：
```bash
docker compose restart backend
```

---

## 🌐 访问应用

启动完成后，在浏览器访问：

- **前端主页**: http://localhost:5000
- **图谱页面**: http://localhost:5000/graph
- **文档上传**: http://localhost:5000/upload
- **后端 API**: http://localhost:8000
- **Neo4j 浏览器**: http://localhost:7474
- **AstrBot 后台**: http://localhost:6125

---

## 🛑 停止服务

### 停止 Docker 服务
```bash
cd fire-eye-system
docker compose down
```

---

## ⚠️ 常见问题

### 问题1：端口被占用
本项目 Docker 默认端口为：
- 前端：5000
- 后端：8000
- Neo4j：7474 / 7687
- AstrBot：6125

如果端口冲突，请修改 `docker-compose.yml` 后重新执行：
```bash
docker compose up -d --build
```

### 问题2：浏览器显示旧样式
- 按 `Ctrl + Shift + R` 强制刷新
- 或使用无痕模式打开

### 问题3：后端连接失败
检查 Docker 容器是否运行：
```bash
docker ps
```

查看后端日志：
```bash
docker logs fire-eye-backend --tail 200
```

如果是 Neo4j 未就绪导致失败，执行：
```bash
docker compose restart backend
```

### 问题4：前端容器异常
查看前端日志：
```bash
docker logs fire-eye-frontend --tail 200
```

## 🎯 开发建议

1. **统一使用 Docker 管理服务**：
   - 不再使用本地 `start-backend.bat` 启动后端
   - 不再混用宿主机 Python 进程和容器内后端
   - 所有服务统一通过 `docker compose` 管理

2. **修改代码后**：
   - 如需重新构建镜像：`docker compose up -d --build`
   - 如仅需重启后端：`docker compose restart backend`
   - 前端页面修改后若未同步生效，可重启前端容器

3. **查看日志**：
   - 后端日志：`docker logs fire-eye-backend -f`
   - 前端日志：`docker logs fire-eye-frontend -f`
   - Neo4j 日志：`docker logs fire-eye-neo4j -f`
