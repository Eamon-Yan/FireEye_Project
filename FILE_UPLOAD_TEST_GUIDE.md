# 文件上传功能测试指南

## 测试准备

### 1. 确认系统状态
```bash
# 检查所有容器是否运行
docker ps

# 应该看到:
# - astrbot (运行中)
# - fire-eye-backend (运行中)
# - fire-eye-neo4j (运行中)
# - fire-eye-redis (运行中)
```

### 2. 准备测试文件
准备以下测试文件:
- **小文件**: 测试文档.txt (< 1MB)
- **中等文件**: 火灾调查报告.pdf (3-5MB)
- **大文件**: 完整报告.docx (8-10MB)
- **无效文件**: 图片.jpg (用于测试错误处理)

---

## 测试用例

### 测试 1: 基本上传流程（TXT文件）

**步骤**:
1. 在 Telegram 中发送命令: `/火瞳上传`
2. 发送一个小的 TXT 文件

**预期结果**:
```
Bot: 📤 正在下载文件...
Bot: 📤 正在上传到服务器...
Bot: ✅ 文件上传成功！
     🔄 正在分析文档...
     任务ID: task_xxxxx
     请稍候，这可能需要30-60秒...

[等待30-60秒]

Bot: ✅ 文档分析完成！
     
     📊 统计信息:
     • 节点数量: X
     • 关系数量: X
     • 事件链数量: X
     
     🔗 提取的事件链（前5条）:
     ...
```

**验证点**:
- [ ] 文件成功下载
- [ ] 文件成功上传
- [ ] 获得任务ID
- [ ] 显示分析进度
- [ ] 显示最终结果
- [ ] 结果包含事件链

### 测试 11: 会话管理

**验证点**:
- [ ] 会话正确创建
- [ ] 会话正确过期
- [ ] 新会话正确创建

## 快速测试命令

```bash
# 1. 检查插件状态
docker logs astrbot --tail 50 | grep Fire-Eye

# 2. 检查后端状态
docker logs fire-eye-backend --tail 50

# 3. 检查临时文件
docker exec astrbot ls -la /tmp/fire-eye/

# 4. 清理临时文件
docker exec astrbot rm -rf /tmp/fire-eye/*

# 5. 重启插件
docker restart astrbot
```
