### 2. ✅ 异步任务轮询器 (`api/polling.py`)
**功能**:
- 后台轮询任务状态
- 每3秒检查一次任务状态
- 最长轮询5分钟
- 支持进度回调
- 完成/失败通知

**关键方法**:
- `poll_task()` - 轮询任务直到完成或超时

**轮询流程**:
```
开始轮询
  ↓
每3秒检查状态
  ↓
状态 = pending/processing → 继续轮询
状态 = completed → 调用 on_complete 回调
状态 = failed → 调用 on_failed 回调
超时（5分钟）→ 调用 on_failed 回调
```

## 技术细节

### 文件处理流程
1. **验证阶段**:
   - 检查文件类型（.pdf, .txt, .docx）
   - 检查文件大小（<10MB）

2. **下载阶段**:
   - 从IM平台URL下载文件
   - 保存到临时目录（/tmp/fire-eye）
   - 生成唯一文件名（UUID）

3. **上传阶段**:
   - 使用 multipart/form-data 格式
   - 包含文件流和session_id
   - 发送到 `/api/v1/chat/upload`

4. **清理阶段**:
   - 上传成功后立即删除临时文件
   - 保护敏感数据

### 异步任务处理
1. **任务创建**:
   - 后端立即返回 task_id
   - 状态: pending

2. **后台处理**:
   - 提取文本内容
   - LLM分析提取事件链
   - 创建图节点和关系
   - 更新进度（10% → 50% → 90% → 100%）

3. **状态轮询**:
   - 插件每3秒查询一次
   - 检查状态变化
   - 超时时间: 5分钟

4. **结果通知**:
   - 完成: 显示提取的事件链
   - 失败: 显示错误信息
   - 超时: 提示手动查询

### 会话管理
- **会话ID**: UUID格式
- **TTL**: 30分钟
- **存储**: 内存字典
- **清理**: 自动清理过期会话
