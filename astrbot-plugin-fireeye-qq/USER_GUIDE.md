# 火瞳 QQ 插件用户指南

欢迎使用火瞳 QQ 插件！本指南将帮助你快速上手。

## 快速开始

### 基本命令

火瞳插件提供以下基本命令:

| 命令 | 别名 | 功能 | 示例 |
|------|------|------|------|
| `/火瞳查询` | `/ht查询` | 查询火灾事件 | `/火瞳查询 3.21火灾` |
| `/火瞳统计` | `/ht统计` | 查看统计信息 | `/火瞳统计` |
| `/火瞳帮助` | `/ht帮助` | 显示帮助 | `/火瞳帮助` |

### 发送命令

在 QQ 中直接输入命令:

```
/火瞳查询 火灾事故
```

机器人会在几秒内返回结果。

## 详细使用说明

### 1. 查询命令

#### 基本查询

查询知识图谱中的信息:

```
/火瞳查询 <关键词>
```

**示例:**

```
/火瞳查询 3.21火灾事故
/火瞳查询 电气隐患
/火瞳查询 消防设施缺陷
```

#### 查询结果解读

查询结果包含以下信息:

```
🔍 查询结果 (3 条)

1. [FireEvent] 3.21某商场火灾事故
   2023年3月21日发生的火灾事故
   └─ CAUSED_BY → [Hazard] 电线老化 (92%)
   └─ RESULTED_IN → [Consequence] 人员伤亡 (85%)

2. [Hazard] 电气隐患
   老旧电线和电气设备
   └─ LOCATED_IN → [Location] 某商场 (78%)

3. [Consequence] 人员伤亡
   火灾导致的人员伤亡
   └─ CAUSED_BY → [FireEvent] 3.21某商场火灾事故 (85%)
```

**字段说明:**

- `[类型]`: 节点类型（FireEvent=火灾事件, Hazard=隐患, Consequence=后果）
- `描述`: 节点的详细描述
- `关系`: 与其他节点的关系（CAUSED_BY=导致, RESULTED_IN=导致, 等）
- `百分比`: 关系的置信度

#### 查询技巧

1. **精确查询**: 使用具体的事件名称或日期
   ```
   /火瞳查询 2023年3月21日火灾
   ```

2. **模糊查询**: 使用关键词
   ```
   /火瞳查询 火灾
   /火瞳查询 隐患
   ```

3. **多词查询**: 使用多个关键词
   ```
   /火瞳查询 电气隐患 火灾
   ```

## 高级功能

### 会话管理

系统会自动为每个用户创建会话，记录对话上下文。

**会话特性:**

- 自动创建: 第一次使用时自动创建
- 自动过期: 30分钟无活动后自动过期
- 上下文保存: 记录最近的查询和结果

**示例:**

```
用户: /火瞳查询 3.21火灾事故
机器人: [返回查询结果]

用户: /火瞳查询 主要原因
机器人: [理解为查询3.21火灾事故的主要原因]
```

### 多轮对话

支持连续的多轮对话:

```
用户: /火瞳查询 火灾事故
机器人: [返回火灾事故列表]

用户: /火瞳查询 第一个事故的原因
机器人: [返回第一个事故的原因]

用户: /火瞳查询 相关的隐患
机器人: [返回相关隐患]
```

## 常见问题

### Q: 查询没有返回结果怎么办？

**A:** 可能的原因:

1. 知识图谱中没有相关数据
2. 查询关键词不匹配
3. 后端服务异常

**解决方案:**

- 尝试使用不同的关键词
- 检查关键词拼写
- 稍后重试
- 联系管理员

### Q: 查询响应很慢怎么办？

**A:** 可能的原因:

1. 系统负载过高
2. 网络延迟
3. 查询结果过多

**解决方案:**

- 稍后重试
- 使用更具体的查询条件
- 检查网络连接

### Q: 如何获取更详细的信息？

**A:** 

1. 使用更具体的查询条件
2. 查询相关的节点
3. 查看统计信息

### Q: 会话过期了怎么办？

**A:** 

会话过期后会自动创建新会话，无需手动操作。

### Q: 可以查询哪些类型的信息？

**A:** 

可以查询以下类型的信息:

- **火灾事件** (FireEvent): 具体的火灾事故
- **隐患** (Hazard): 火灾的原因和隐患
- **后果** (Consequence): 火灾的影响和后果
- **位置** (Location): 火灾发生的地点
- **时间** (Time): 火灾发生的时间

## 最佳实践

### 1. 查询建议

- ✅ 使用具体的事件名称或日期
- ✅ 使用相关的关键词
- ❌ 避免过于宽泛的查询
- ❌ 避免使用特殊字符

### 2. 命令使用

- ✅ 使用 `/ht` 作为快捷方式
- ✅ 在群组中使用命令
- ✅ 等待响应后再发送新命令
- ❌ 不要频繁发送相同的查询

### 3. 结果解读

- ✅ 注意置信度百分比
- ✅ 查看相关的节点关系
- ✅ 参考多个查询结果
- ❌ 不要过度依赖单一结果
